# Agentic OS - AI Agent Guidelines

Welcome to the Agentic OS repository! This document (`AGENTS.md`) outlines the essential context, boundaries, and best practices for AI agents interacting with this codebase.

## 1. System Architecture
- **Core Reasoning**: `agent_core/` handles the ReAct loops, context management, and sandbox execution. It is the brain.
- **Abilities**: Modules in `agent_skills/`, `skills/`, `agent_productivity/`, etc.
- **Memory & RAG**: `agent_memory/` and `agent_rag/` back the system `pgvector` dependencies.
- **Routing**: `llm_router/` and `rl_router/` manage dynamic bandwidth and decision-making depth.
- **Entry Points**: 
  - `gateway/main.py` is the main Python CLI and Server.
  - `ui/app.py` is the Streamlit interface.

## 2. Coding Standards
- **Typing**: Use strong Python 3.11+ type hints (`typing` module) everywhere. Run `pyright` to verify strict correctness before committing.
- **Linting & Formatting**: Follow `PEP 8` standard spacing and indentation.
- **Error Handling**: Use structured logs rather than raw `print` statements in production components. Be explicit about failure boundaries, specifically when wrapping external tools in sandboxes.
- **Tests**: Write `pytest` compatible asynchronous tests for new features.

## 3. Pull Request & Commit Rules
- Commit messages must follow Conventional Commits (e.g., `feat: xyz`, `fix: abc`, `chore: ...`).
- Only push to feature branches, never directly to `main`.
- For complex changes, document steps clearly in PR descriptions.

## 4. Danger Zones
- **`agent_core/tools.py`**: Risky subprocess sandboxing. DO NOT edit this file without verifying security constraints (e.g., `WorkspaceManager`).
- **`gateway/server.py`**: This connects the WebSocket events to the React Loop. Avoid blocking the event loop or removing high-timeout HTTP interactions unless explicitly required.
- **`rl_router/`**: The state spaces and bandit models here are highly tuned. Only touch feature vectors or reward scales if an explicit goal mentions Reinforcement Learning optimizations.

## 5. Test Expectations
- Local testing uses a virtual environment `.venv`.
- Always run `python -m pytest tests/` before claiming a task is fully complete.
- Ensure any mocked resources (like `sqlite3`, `patch`) correctly clean up their scopes.
